## 2. Demo 创作思路

### 灵感来源

我住的小区有 2000 多户，但邻里关系几乎是断裂的：
- 有次家里水管爆了，物业下班联系不上，最后是楼下退休的张师傅帮忙修的——但我之前完全不知道他会修水管；
- 楼上王阿姨经常包多了饺子想分享，但又不好意思挨家挨户敲门；
- 社区里有位独居的李奶奶，子女在外地，她想找年轻人陪她去趟医院都不会用 App 发需求。

这些真实的生活片段，就是邻里圈的雏形。

### 想解决的问题

| 用户群体 | 真实痛点 |
|---------|---------|
| 独居青年 | 想社交但没有场景，想求助找不到人 |
| 空巢老人 | 子女不在身边，就医、采买、陪伴无人帮忙 |
| 带娃家庭 | 临时看护难找，隔代教育缺资源 |
| 有技能的人 | 想分享技能但没有平台，价值无法变现 |
| 全体居民 | 应急时刻只能靠物业 / 110，邻里资源被闲置 |

### 为什么做这个方向

1. **社会价值真实**：这不是"为了做而做"的需求，是每天都在发生的真实痛点。中国有 4 亿城市社区人口，邻里互助是被严重低估的场景。
2. **AI 能真正发力**：传统互助平台卡在"匹配效率低"——关键词匹配找不到"会修水管的张师傅"。语义匹配 + 距离 + 信誉三维模型，让 AI 在这个场景有了真实价值。
3. **差异化清晰**：市面上的社区产品要么是物业工具（解决缴费报修），要么是兴趣社交（解决聊天），**没有人把"互助"做成生态**。邻里圈用"一个平台四种连接"切入了这个空白。

```
Step 1: 创意构思 → 用自然语言描述想法，TRAE 辅助梳理为 project-spec.md
```

#### 步骤 2：数据库迁移与后端服务搭建

最让我印象深刻的是**时间银行转账的事务安全**——我只说了"实现家庭时间池转账"，TRAE 直接给出了带行锁的完整实现：

```typescript
// TRAE 生成的事务安全转账逻辑（关键片段）
await client.query('BEGIN');
try {
  // 行锁防止并发扣减
  const fromAccount = await client.query(
    'SELECT * FROM time_accounts WHERE user_id = $1 FOR UPDATE',
    [fromUserId]
  );
  // 最低保留余额校验
  if (fromAccount.balance < amount + MIN_RETAIN) {
    throw new Error('余额不足');
  }
  // 扣减 + 增加（同一事务内）
  await client.query('UPDATE time_accounts SET balance = balance - $1 WHERE user_id = $2', [amount, fromUserId]);
  await client.query('UPDATE time_accounts SET balance = balance + $1 WHERE user_id = $2', [amount, toUserId]);
  // 幂等键防重复
  await client.query('INSERT INTO time_transactions (idempotency_key, ...) VALUES ($1, ...) ON CONFLICT DO NOTHING', [idempotencyKey]);
  await client.query('COMMIT');
} catch (e) {
  await client.query('ROLLBACK');
  throw e;
}
```

#### 步骤 3：AI 三维智能匹配（核心创新）

这是项目最核心的创新点。我向 TRAE 描述："匹配要考虑语义相似度、地理距离、信誉分三个维度"，TRAE 设计了完整的评分公式和降级策略：

```
totalScore = matchScore × 0.5 + (reputation/5) × 0.3 + distanceScore × 0.2
```

- **语义层**：调用大模型 API（兼容 OpenAI / 通义千问 / 智谱），输出 0-1 匹配度
- **距离层**：PostgreSQL POINT 类型 + Haversine 公式，10km 内线性衰减
- **信誉层**：近 50 条评价均值 + 时间衰减因子
- **降级**：LLM 不可用时自动降级为类别匹配，保证 100% 可用

#### 步骤 4：应急邻里 WebSocket 实时推送

我描述："用户发起应急求助后，300 米内的响应者要能实时收到推送"，TRAE 帮我实现了：
- WebSocket 连接管理 + 心跳保活
- 地理围栏检索（PostgreSQL `<->` 球面距离运算符）

#### 步骤 5：前端开发与样式优化

#### 步骤 6：部署运维

TRAE 帮我生成了完整的 Docker Compose 编排 + Nginx 配置 + PM2 部署脚本，一键部署到生产服务器。
