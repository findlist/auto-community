### P0 - 严重（生产代码 Bug）

#### 1. CreateService.tsx - React Hooks 条件调用（运行时崩溃风险）

- **文件**：[CreateService.tsx](file:///e:/work/auto-community/client/src/pages/TimeBank/CreateService.tsx)
- **行号**：第 34、40、69 行
- **lint 规则**：`react-hooks/rules-of-hooks`
- **问题描述**：

  第 34 行存在 early return `if (!isAuthenticated) return null;`，而 `useMemo`（第 40 行）和 `useFormValidation`（第 69 行）在该 return **之后**调用。React Hooks 规则要求 Hooks 必须在每次渲染中以完全相同的顺序调用，条件性 return 会跳过后续 Hooks，导致 React 抛出运行时错误：

  ```
  React Hook "useMemo" is called conditionally. React Hooks must be called in the exact same order in every component render.
  React Hook "useFormValidation" is called conditionally. React Hooks must be called in the exact same order in every component render.
  ```

- **影响**：当用户未登录访问创建服务页时，组件会因 Hooks 顺序不一致而崩溃。
- **修复建议**：将所有 Hooks 调用移到 early return 之前，或将 early return 逻辑改为条件渲染（不提前返回）。
