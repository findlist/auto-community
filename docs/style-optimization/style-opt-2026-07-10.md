## 具体修改内容

### 1. Auth 模块：统一玻璃态暗色风格

**问题**：Auth 模块四个页面风格严重不统一——
- `Login.tsx`：精致的玻璃态暗色风格（背景图 + 暗化遮罩 + 玻璃态卡片 + 图标输入框）
- `Register.tsx`：亮色白底卡片风格，与 Login 完全割裂
- `ForgotPassword.tsx`：最基础的 Tailwind gray 色表单，无图标、无卡片容器、无动画
- `ResetPassword.tsx`：同上，最基础风格

**优化**：将 Register、ForgotPassword、ResetPassword 统一为与 Login 一致的玻璃态暗色风格——
- 暗化遮罩背景（`bg-neutral-900/55 backdrop-blur-sm`）
- 玻璃态表单卡片（`bg-white/10 backdrop-blur-xl border border-white/20 rounded-2xl`）
- 图标前缀输入框（Phone / Lock / User / KeyRound 等 lucide 图标）
- 统一的白色文字 + 半透明占位符
- 统一的加载态 spinner + 箭头按钮
- 统一的品牌字标 + tracking-wide 副标题
- 统一的 `animate-fade-in-up` 进场动画
- 统一的底部标语"AI 智能推荐 · 三维匹配 · 温暖社区"
- 成功状态页同样使用暗色背景 + 玻璃态成功标记，避免状态切换时布局跳变

### 2. Admin 模块：接入设计系统 neutral 色阶

**问题**：Admin 模块 11 个源文件中共 366 处使用 Tailwind 默认 `gray-数字` 色阶，未使用项目设计系统的 `neutral` 色阶，导致后台与前台存在色差（gray 偏冷蓝调，neutral 为纯灰阶），且未遵循 `index.css` 中定义的设计 Token。

**优化**：将所有 `gray-数字` 替换为 `neutral-数字`，色阶一一对应——

| 原 Tailwind gray | 替换为 neutral | 语义对应 |
|-----------------|---------------|---------|
| gray-50 | neutral-50 | 背景 |
| gray-100 | neutral-100 | 次背景 / hover |
| gray-200 | neutral-200 | 边框 |
| gray-300 | neutral-300 | 强边框 |
| gray-400 | neutral-400 | 三级文字 |
| gray-500 | neutral-500 | 二级文字 |
| gray-600 | neutral-600 | 二级文字 |
| gray-700 | neutral-700 | 一级文字 |
| gray-800 | neutral-800 | 标题文字 |
| gray-900 | neutral-900 | 最强文字 |

**替换统计**：

| 文件 | 替换处数 |
|------|---------|

| UserManagement.tsx | 38 |
| ContentReview.tsx | 61 |
