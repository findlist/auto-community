## 具体修改内容

### 1. 空状态统一：emoji 替换为 lucide 图标容器

**问题**：TimeBank、SharedKitchen、SkillExchange 三个模块的空状态使用 emoji（🕐🍽️🔧）作为视觉占位。emoji 存在以下缺陷：
- 跨平台渲染不一致（iOS/Android/Windows 表现各异）
- 与项目设计系统基于 lucide 图标的整体风格割裂
- 无法继承 `currentColor`，无法随主题色调变化
- 视觉重量偏大，与编辑式列表的克制风格不协调

**优化**：将三个模块的空状态 emoji 替换为统一的「lucide 图标 + 圆形底色容器」模式——
- 图标容器：`w-12 h-12 rounded-full bg-neutral-100`，内嵌对应模块语义的 lucide 图标
- 图标颜色：`text-neutral-400`（与空状态文字色阶一致）
- TimeBank 错误状态使用 `bg-red-50` + `text-red-400` 区分语义

| 模块 | 原 emoji | 替换为 lucide 图标 |
|------|---------|-------------------|
| TimeBank 空列表 | 🕐 | `Clock` |
| TimeBank 错误 | 无图标（纯文字） | `Clock`（红色态） |
| SharedKitchen | 🍽️ | `ChefHat` |
| SkillExchange | 🔧 | `Wrench` |

### 2. Admin Dashboard 统计卡片：圆角对齐设计规范 + hover 升降反馈

**问题**：
- 统计卡片使用 `rounded-2xl`（16px），但设计规范明确规定卡片应使用 `--radius-lg`（12px，即 `rounded-xl`），小卡片用 16px 圆角显得偏柔、不够利落
- 卡片 hover 仅有阴影变化（`hover:shadow-md`），缺少位移反馈，交互感不足

**优化**：
- 统计卡片 `rounded-2xl` → `rounded-xl`，对齐设计规范 `--radius-lg`
- 图表错误容器同步 `rounded-2xl` → `rounded-xl`，保持一致
- 新增 `hover:-translate-y-0.5`，hover 时卡片轻微上浮 2px，增强可交互感知

### 3. AdminLayout 移动端抽屉：滑入动画

**问题**：移动端侧边栏抽屉打开时无进场动画，瞬间出现，体验生硬。

**优化**：为移动端抽屉 `aside` 添加 `animate-slide-in-right` 类（复用 `index.css` 中已定义的 `slideInRight` 关键帧动画），抽屉从左侧滑入，过渡自然。

### 4. TimeBank 快捷入口：边框定义强化 + hover 升降反馈

**问题**：
- 快捷入口卡片边框使用 `border-neutral-100`（`#f5f5f5`），在白色背景下几乎不可见，卡片边界模糊
- hover 仅有阴影和边框色变化，缺少位移反馈

**优化**：
- `border-neutral-100` → `border-neutral-200`（`#e5e5e5`），边框清晰可辨
- 新增 `hover:-translate-y-0.5`，与 Admin Dashboard 卡片反馈一致
