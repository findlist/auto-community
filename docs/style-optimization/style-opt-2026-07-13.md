## 三、具体修改内容

### 1. 模块列表页深化模块色身份（核心优化）

**问题**：07-12 已将眉题色改为模块色，但列表项内部的「悬停标题色 / 价格色 / 分类色 / 进度条色」仍统一使用 `emerald-700` / `emerald-600`，导致三个模块列表页在视觉上几乎不可区分，模块身份仅靠眉题一个点位支撑，过弱。

**修改思路**：将模块色从眉题单点位扩展到「悬停标题 + 价格 + 分类 + Tab 下划线 + 进度条」多点位的色彩系统，让用户在列表区也能立即识别当前模块。

### 2. ServiceCard 类型色贯穿 5 个点位

**问题**：ServiceCard 中「提供」类型用 emerald、「需求」类型用 violet 的色彩约定仅体现在竖条与类型标签两处，而「标题悬停色 / 分类文本色 / 箭头色」三个交互点位仍统一使用 emerald 或 neutral-900，导致需求类型卡片的色彩断裂——竖条是紫，但悬停标题变绿，箭头又变黑，视觉不统一。

**修改**：将类型色抽象为变量 `typeTextHover` / `typeTextColor` / `typeArrowColor`，根据 `service.type` 在 emerald（提供）与 violet（需求）之间切换，让类型色贯穿「竖条 / 类型标签 / 标题悬停 / 分类文本 / 箭头」5 个点位。同时为竖条增加 `group-hover:h-14` 高度变化，悬停时竖条从 `h-12` 拉长到 `h-14`，作为额外的微交互反馈。

| 元素 | 优化前 | 优化后（provide / request） |
| --- | --- | --- |
| 竖条高度 | `h-12`（静态） | `h-12` → `group-hover:h-14`（悬停拉长） |
| 标题悬停色 | 统一 `group-hover:text-emerald-700` | `group-hover:text-emerald-700` / `group-hover:text-violet-700` |
| 分类文本色 | 统一 `text-emerald-700/80` | `text-emerald-700/80` / `text-violet-700/80` |
| 箭头色 | 统一 `text-neutral-900` | `text-emerald-600` / `text-violet-600` |

**涉及文件**：[TimeBank/ServiceCard.tsx](file:///e:/work/auto-community/client/src/pages/TimeBank/ServiceCard.tsx)

### 3. Auth 页面焦点环细化与卡片悬停反馈

**问题**：
1. 4 个认证页面（Login / Register / ForgotPassword / ResetPassword）的输入框焦点环为 `focus:ring-white/30`，与背景对比度偏弱，无障碍体验可改进。
2. 表单卡片仅有静态阴影，悬停时无任何反馈，缺少「卡片是可交互容器」的视觉提示。
3. 输入框左侧图标缺少 `transition-colors`，焦点态切换时图标颜色瞬变，不够柔和。

**修改**：
1. 所有输入框焦点环从 `focus:ring-white/30` 提升到 `focus:ring-white/40 focus:border-white/40`，同时强化环透明度与边框色，焦点态更易识别。
2. 表单卡片增加 `transition-shadow duration-300 hover:shadow-[0_25px_60px_-15px_rgba(0,0,0,0.5)]`，悬停时阴影加深，营造轻微的「浮起」感。
3. 所有输入框图标添加 `transition-colors`，为未来的图标焦点态变化预留过渡基础。

**涉及文件**：
- [Auth/Login.tsx](file:///e:/work/auto-community/client/src/pages/Auth/Login.tsx)
- [Auth/Register.tsx](file:///e:/work/auto-community/client/src/pages/Auth/Register.tsx)
- [Auth/ForgotPassword.tsx](file:///e:/work/auto-community/client/src/pages/Auth/ForgotPassword.tsx)
- [Auth/ResetPassword.tsx](file:///e:/work/auto-community/client/src/pages/Auth/ResetPassword.tsx)

### 4. Admin 侧边栏导航分组与微交互

**问题**：Admin 侧边栏原有 11 个导航项扁平排列在同一组中，无分类信息，管理员扫读目标项需要纵向扫描整列。同时导航项悬停仅有背景色变化，缺少位移反馈。

**修改**：
1. 将 11 个导航项按职能分为 3 组：
   - **运营**：Dashboard、用户管理、内容审核、订单管理、举报处理、实名认证（6 项）
   - **数据**：效果度量、A/B 测试（2 项）
   - **系统**：首页图片、操作日志、系统配置（3 项）
2. 每组上方添加极小号大写字母组标题（`text-[10px] font-mono tracking-widest uppercase`），编辑式分隔，组间用 `border-t border-neutral-100` 分割。
3. 导航项悬停增加 `hover:translate-x-0.5` 微位移，激活态从 `transition-colors` 升级为 `transition-all duration-200`，让位移过渡顺滑。
4. 图标尺寸从 `w-5 h-5` 调整为 `w-[18px] h-[18px]`，与 14px 文字基线更协调，减少图标视觉权重过重。
5. 导航项垂直间距从 `gap-1` 调整为组内 `gap-0.5`、组间 `mt-4 pt-4`，让分组层级更清晰。

**涉及文件**：[Admin/AdminLayout.tsx](file:///e:/work/auto-community/client/src/pages/Admin/AdminLayout.tsx)

### 5. Dashboard 统计卡片悬停微提

**问题**：Dashboard 顶部 7 个统计卡片悬停时仅阴影从 `shadow-sm` 变为 `shadow-md`，变化幅度过小，用户难以感知到「卡片可交互」的暗示。

**修改**：在保留原有阴影变化的基础上，增加 `hover:-translate-y-0.5`，让卡片悬停时向上微提 2px，配合阴影变化形成「浮起」效果，反馈更明确。

**涉及文件**：[Admin/Dashboard.tsx](file:///e:/work/auto-community/client/src/pages/Admin/Dashboard.tsx)

## 四、优化前后对比说明

### 4.1 模块身份辨识度（核心提升）

| 维度 | 优化前 | 优化后 |
| --- | --- | --- |
| 模块色应用点位 | 仅眉题 1 个 | 眉题 + Tab 下划线 + 悬停标题 + 价格 + 分类 + 进度条（5-6 个） |
| SkillExchange 悬停 | 标题变 emerald-700（与模块无关） | 标题变 blue-700（对齐模块色） |
| SharedKitchen 悬停 | 标题/价格/进度条变 emerald（与模块无关） | 全部变 orange（对齐模块色） |
| TimeBank Tab 下划线 | neutral-900（与模块无关） | violet-600（对齐模块色） |
| ServiceCard 需求类型 | 竖条紫，但悬停标题变绿、箭头变黑 | 竖条/标签/悬停/分类/箭头统一紫 |

**效果**：用户在三个模块列表页之间切换时，从「仅眉题提示模块」升级为「整个列表区的色彩系统都在提示模块」，模块身份辨识度大幅提升，emerald 不再是无处不在的默认色，而是回归为「提供/成功/正向」的语义色。

### 4.2 Auth 焦点态与卡片反馈

| 维度 | 优化前 | 优化后 |
| --- | --- | --- |
| 输入框焦点环 | `ring-white/30`（对比度弱） | `ring-white/40 border-white/40`（环+边框双强化） |
| 表单卡片悬停 | 静态阴影，无反馈 | 阴影加深，模拟浮起 |
| 图标焦点过渡 | 瞬变 | `transition-colors` 预留过渡基础 |

**效果**：4 个认证页面（Login / Register / ForgotPassword / ResetPassword）的输入焦点态更易识别，尤其对键盘导航用户与低视力用户更友好；卡片悬停反馈让用户感知到表单是可交互容器，而非静态展示。

### 4.3 Admin 信息架构

| 维度 | 优化前 | 优化后 |
| --- | --- | --- |
| 导航项数量 | 11 项扁平排列 | 3 组（6 + 2 + 3）分组排列 |
| 组标题 | 无 | 极小号大写字母 + 字距，编辑式 |
| 导航项悬停 | 仅背景色 | 背景色 + 0.5px 右移 |
| 图标尺寸 | w-5 h-5（20px，偏大） | w-[18px] h-[18px]（与 14px 文字协调） |

**效果**：管理员扫读侧边栏时，可先定位到「运营/数据/系统」分组，再在组内找具体功能，扫读路径从「线性扫描 11 项」变为「树状扫描 3 组」，效率提升。组标题的编辑式排版与前台页面的眉题风格保持一致，前后台视觉语言统一。

### 4.4 Dashboard 卡片反馈

| 维度 | 优化前 | 优化后 |
| --- | --- | --- |
| 悬停反馈 | 仅阴影 sm → md | 阴影 sm → md + 上提 2px |

**效果**：统计卡片悬停时「浮起」效果明确，用户感知到卡片是可交互的（即使目前无点击行为，也为未来增加点击下钻留出交互空间）。
