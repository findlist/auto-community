## 三、具体修改内容

### 1. 全局基础样式补齐（index.css）

**问题**：
1. 文本选区（`::selection`）使用浏览器默认蓝底白字，与"邻里绿"品牌主色脱节，选中文本时无品牌识别度。
2. `html` 缺 `scroll-behavior: smooth`，同页锚点跳转生硬。
3. 缺 `text-rendering: optimizeLegibility`，标题与正文在小字号下字形与字距不够清晰。
4. 移动端点击高亮（`-webkit-tap-highlight-color`）未关闭，按钮点击有灰色闪斑。
5. `prefers-reduced-motion` 偏好下未禁用平滑滚动，与"减弱动画"语义冲突。

**修改**：在 `html` 规则中补齐 `scroll-behavior: smooth`、`text-rendering: optimizeLegibility`、`-webkit-tap-highlight-color: transparent`；新增 `::selection` / `::-moz-selection` 规则使用 `--color-primary-200` 底 + `--color-primary-900` 字；在 `prefers-reduced-motion` 媒体查询中追加 `html { scroll-behavior: auto !important }`。

| 元素 | 优化前 | 优化后 |
| --- | --- | --- |
| 文本选区 | 浏览器默认蓝底白字 | 邻里绿底（`--color-primary-200`）+ 深绿字（`--color-primary-900`） |
| 平滑滚动 | 无 | `scroll-behavior: smooth`（减弱动画偏好下自动禁用） |
| 字形渲染 | 默认 | `text-rendering: optimizeLegibility` |
| 移动端点击高亮 | 系统灰色闪斑 | `transparent`（关闭） |

**涉及文件**：[index.css](file:///e:/work/auto-community/client/src/index.css)

### 2. Auth 4 页面输入框焦点柔光晕 + 密码切换命中区增强

**问题**：
1. 4 个 Auth 页面的玻璃态输入框焦点态为 `focus:ring-2 focus:ring-white/40`，仅有 2px 白色描边环，在暗化背景上焦点指向偏弱，缺少"光晕"层次。
2. 登录/注册的密码显隐切换按钮为 `right-3.5` 裸图标，命中区仅 16×16px，且 `hover:text-white/80` 无背景反馈，移动端点击易误触、桌面端悬停无"可点"暗示。

**修改**：
1. 将输入框焦点态统一升级为 `focus:ring-white/40 focus:border-white/50 focus:shadow-[0_0_0_4px_rgba(255,255,255,0.06)]`——保留 2px 白环（无障碍），叠加 4px 半透明白色光晕（视觉层次），形成"描边环 + 外发光"双层焦点。
2. 密码切换按钮改为 `right-2 p-1.5 rounded-lg hover:text-white hover:bg-white/10 transition-all`——通过 `p-1.5` 扩大命中区至 28×28px，`right-2` 配合 padding 使图标视觉中心不变（仍为距右 22px），悬停时圆角底色浮现，明确"可点击"。

| 元素 | 优化前 | 优化后 |
| --- | --- | --- |
| 输入框焦点 | `focus:ring-2 focus:ring-white/40 focus:border-white/40` | `focus:ring-2 focus:ring-white/40 focus:border-white/50 focus:shadow-[0_0_0_4px_rgba(255,255,255,0.06)]` |
| 密码切换按钮 | `right-3.5 text-white/50 hover:text-white/80 transition-colors`（16×16） | `right-2 p-1.5 rounded-lg text-white/50 hover:text-white hover:bg-white/10 transition-all`（28×28，图标位置不变） |
